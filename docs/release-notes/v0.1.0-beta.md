# Desktop Portal 0.1.0-beta 发布说明

发布日期：2026-05-13

## 定位

Desktop Portal 是 Windows 本地快捷切换工具，用全局快捷键快速打开或切换到文件、文件夹、网页和程序。

## 功能

- 规则管理：新增、编辑、删除、启用、禁用。
- 快捷键录入：F1-F12、数字键、常见组合键、鼠标中键、鼠标侧键。
- 全局快捷键：使用 Win32 `RegisterHotKey`。
- 鼠标快捷键：使用低级鼠标 hook，并异步派发执行。
- 目标执行：URL、文件、文件夹、exe。
- URL app 模式：优先 Chrome，其次 Edge。
- 窗口激活：支持窗口标题、进程、资源管理器路径和缓存窗口句柄。
- 系统托盘：关闭到托盘、暂停/恢复快捷键、重载配置、退出。
- 开机自启：使用当前用户 HKCU Run。
- 配置文件：本地 JSON。
- 健康检查：检查启用规则的快捷键格式、快捷键重复和目标有效性。
- 配置备份：保存前自动保留最近 5 份 `config.backup.*.json`。
- 日志：本地 app.log，支持轮转。
- 崩溃兜底：UI、后台任务和 AppDomain 未处理异常会写入本地日志。
- 单实例保护：重复启动会唤起已有实例。

## 本地文件

- 配置：`%AppData%\DesktopPortal\config.json`
- 配置备份：`%AppData%\DesktopPortal\backups\config.backup.*.json`
- 日志：`%AppData%\DesktopPortal\logs\app.log`

## 隐私

不联网，不采集数据，不上传配置，不需要账号。

## 已知限制

- 普通浏览器标签页无法精确切换，只能按窗口标题或域名尽量匹配。
- 未签名构建可能触发 SmartScreen。
- 高权限窗口可能无法被普通权限进程激活。
